Leetcode 53 Maximum Subarray

题目来源

Maximum Subarray

题目分析

第一眼看到，我好像前不久做过相似的题目，应该是CVTE的笔试题目，今天回想，那时候应该是做错了，想得太简单了，再看一看感觉不怎么会做啊。
在一个数组中找和最大的子数组，这个应该是一个最优的问题，奈何不怎么会动态规划。
以一个标志记录子数组开头，看情况而改变开头的位置，如果能找到比较大的加和，那么就记录下来。
Java的最小值表示为Integer.MIN_VALUE.

代码分析

借鉴了别人的方法，sum记录一下子数组从此位置开始的和，如果这个和小于0，那么这个开头的位置就可以改变了，因为加上以上个开头开始的和也只会让结果的和减小，并没有用。所以每当sum < 0，就把之前的加和清空，sum重置为当前位置的数字。之后判断sum与max的大小，择优留下。（markdown的小于号好像有什么作用。）

这方法不错，我没想到。要是我的话可能暴力n方了，还是理智点学学人家的方法。之后再补充动态规划方法，先解决飞行棋的问题。
